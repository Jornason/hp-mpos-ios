mPOS for iOS User manual

This document is a user guide for the mPOS application for iOS from Handpoint. In this document you will find information about how to use the mPOS application to conduct secure “chip&pin” transactions and other features of the application. The first section contains a quick start guide, the second section contains information navigation and various settings, third sections explains how to perform transactions and the last section covers how to view transaction information.

Quick start

This section covers how to quickly starting to use the mPOS application to perform transactions. Start by swiping the main screen to the right to access the settings page. Press the Discover button to scan for available card readers, when the scanning is finished the user can select the desired card reader by selecting it’s name from the list. Then the application should connect to the reader and return to the setting menu. From the settings menu the user can swipe back to the left to access the main pay window. To perform a transaction, the user can input the desired amount using the keypad and add a description or image if desired, and then press the orange Pay button to start the transaction. This should start the transaction and display a status display. When the transaction is finished the application will display a receipt for the transaction that can be sent to the user as an e-mail attachment. For more information refer to the rest of this user manual.

Navigation and settings

In this section of the user manual you will find information about how to navigate the application and detailed information about the settings.

Navigation

In this chapter you can find information about navigating the mPOS application. The application main window consists of three main views, that the user can slide between with your finger. The applications starts with displaying the main main payment window, that is in the center. Then the user can slide either to the left or right. If the user slides to the left, he will see a list of older transactions, if there are any. If the users slides to the right he will see a settings menu. If the user selects one of the setting menu, he will enter the desired settings and a navigation bar will be displayed at the top with a “back” button. The user can return to the settings overview by pressing the “back” button.

Settings

In this chapter you can find information about the various settings available in the mPOS application.

Discover (Connect to reader)

To connect to a card reader the user presses the Discover button on the settings page. When the users enters the discover menu the application automatically starts scanning for available readers. Available readers are displayed in a list, where the user can select a reader to connect to by touching the readers name. When the user selects a reader the applications tries to connect to the reader and then displays a message that indicate whether the connection was successful or not. If the connections was successful the application will automatically return to the main settings page, but if the connection is unsuccessful it will remain in the discover menu so the user can select another card reader.

Select currency

The user can select the currency he would like to use from a list of available currencies. To access this list the user press the Settings button and there he presses the Select currency button. In the Select currency settings page there will be a list of available currencies which the user can select from. The user selects currency by tapping on it. The application stores the selected currency between application sessions.

Modify merchant name and address

The user can edit the name and address for the merchant which is displayed on the receipts. To access the name and address settings the user press the Settings button and there he presses the Merchant settings button. In the Merchant settings settings page there will be a two text fields where the user can edit the merchant's name and address. These settings are stored by pressing the Save button.

Contact support

The user can contact Handpoint’s support team by various methods. By pressing on the Support button on the settings page the user can call the support hotline number by simply pressing on the phone number, or send an e-mail by pressing on the E-mail us button. The user get to Handpoint’s web page quickly by pressing the button with the website address.

Perform transaction

In this section of the user manual you can find information regarding performing transactions and related actions.

Add image

To add an image to a transaction the user has to press on the button with the camera logo on the main payment screen. Then the application will display two options, Take photo or Photo library. If the user selects take photo he will go into the camera application and can take a picture, which he can then choose to use or retake. Then the user can get back to the payment screen by pressing the back button in the navigation bar at the top. If the user selects the Photo library option the application will open the photo library and allow the user to select a photo to attach to the transaction.

Top

Add description

To add a description to the transaction, the user has to press the text field labeled Description it is located below the amount display. When the user presses the Description text field a keyboard will appear. The user can use this keyboard to enter his description, and then he can dismiss the keyboard when finished by pressing the return key or touching the Handpoint logo at the top.

Pay

To perform a transaction the user can enter the desired amount using the keypad, add his description or image if desired and then press the orange Pay button to start the transaction. A status display should appear, displaying the transactions status. The transaction can be canceled by pressing the Cancel button in the status display. If the Pay button is not visible but a Refund button is visible, go to the settings page and press the Pay button there to switch back to pay mode.

Top

Refund

To perform a refund transaction the user can go to the settings page and press the Refund button. This will change the mode to the refund mode. Now the user can return to the main pay window, where the Pay button has been replaced with a Refund button. There the user can input the amount, description or add an image just like with the pay transaction, and then press the Refund button to start the refund transaction. A status display should appear, displaying the transaction status. The transaction can be canceled by pressing the Cancel button in the status display. If the Refund button is not visible but a Refund button is visible, go to the settings page and press the Refund button there to switch back to pay mode.

Top

Send receipt

When a transaction is finished, the application should display a receipt. That receipt can be sent to the customer as an PDF file via E-mail. To send the receipt the user can press the text field labeled E-mail address and a keyboard will appear. The user can enter the customer e-mail address and press the Send button. Then the application will create an e-mail message using the native iOS e-mail application, with the customers e-mail address and the receipt as an attachment. The the user can either add something to the message body or just press Send. Then the user is returned to the mPOS application.

Transactions details

In this section of the user manual you can find information about transaction history and related items.

View list of transactions

To view a list of older transactions, the user can either slide the main screen to the left, or he can press the List of records button in the settings page. the list of records shows the date, amount and type of sale for each transaction in a list. The the user can select one transaction by pressing on it in the list. Then the receipt for the selected transaction is displayed. From there the user can both send the receipt as an e-mail attachment or go into the transaction details.

Top

View transaction details

To view the transaction detail the user can press the orange Details button on the receipt. Then the application will display both a detailed customer receipt and a detailed merchant receipt on two separate tabs.

Send receipt

When the user has selected a receipt, that receipt can be sent to the customer as an PDF file via E-mail. To send the receipt the user can press the text field labeled E-mail address and a keyboard will appear. The user can enter the customer e-mail address and press the Send button. Then the application will create an e-mail message using the native iOS e-mail application, with the customers e-mail address and the receipt as an attachment. The the user can either add something to the message body or just press Send. Then the user is returned to the mPOS application.

Additional features

In this section of the user manual you will find information about some additional features, like social network integration that allows the user to post pictures and text to Facebook and Twitter and items catalog. The social network features use the settings from the iOS system, so these have to be set up for these features to work.

Social network integration

The user can share images and text directly from the application to both Facebook and Twitter. To do so the user can go to the Settings page and press on the Social button. From there the user has the option to select a photo or take a new photo using the built in camera. The user can then press either the Tweet or Post to Facebook buttons to post to the relevant social network. The application will then launch the relevant social network window to complete your post.

Items

The mPOS application can store a list of items, which can hold information about the item name or description, price and category. The user can view, edit, add and delete items from this list.

Top

View items

To view the items stored in the application, the user can go to the Settings page and press the Items button. The application will then display a list of items.

Top

Add items

To add an item to the list, the user can go to the list of items as described before and press the + button at the right side of the navigation bar. The the application displays three text fields for the item name, price and category. When the user has entered his information in the text fields he can press the Save button at the right side of the navigation bar to store the item. The application will then return to the list of items.

Top

Edit items

The user can edit an item by going to the list of items and select an item from the list. The application will then display the items properties in the same text fields as in the add items view.

Top

Delete items

The user can also delete an item from the list by going to the list of items and swiping over the item to delete to the left. Then a Delete button will appear on the right side, press the Delete button to delete the item or swipe back to the right to cancel.
