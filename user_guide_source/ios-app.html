Introduction and requerments

Functionality

The Handpoint App and the Handpoint mobile POS device (card reader) enables you to accept chip and pin payments in a secure way. The app provides you with a simple numpad screen where you can enter amounts and initiate a sale. Swiping to the right on the numpad screen shows the Payments History list from where you can view prevous payments. Swiping to the left on the numbad screen brings up the settings view from where you can configure the application.

Card reader

The app connects to a card reader from Handpoint. Please sign up at www.handpoint.com for a card reader if you haven't already. When the sign up procedure has been finished you will recieve a card reader and a license to go with it

Connecting to the card reader is done through the bluetooth settings on your iOS device. First you have to pair the devices and then connect. The Handpoint App automatically connects to the paired device upon startup. A small status bar at the top of the screen shows the status of card reader connection.

License

The license is sent by email as an attachement. To activate the Handpoint app you need to open the license from within your email application. For additional security you are also required to create a pin when you first open the app.

Navigation and settings

Navigation

Navigating the mPOS application is simple. The application main window consists of three main views, which you can move between by sliding with your finger. The applications starts with displaying the main main payment window. Sliding to the left, you will see a list of older transactions. Sliding to the right will show a menu for more actions.

Settings

Pressing this button will bring up the settings view. From there you can change the default currency, edit merchant information for receipts, enter support mode (if requested by Handpoint support staff) and update your card reader.

Refund

Pressing this button will switch the payment method from payment to refund. Transactions will then refund the cardholder for the input amount.

Transactions

Taking payments is simple. On the main view you enter the amount. Optinally you can add a picture and write a description. Then simply press Pay. The card reader handles the rest of it. After the transaction has been processed you will be brought to the receipt view, which details the results of the transaction.

Add image

Pressing the camera button will allow you to take a picture or choose from your photo library to attach to the receipt.

Refund

You can set the app to refund mode, this allows you to refund customers f.ex when returning items previously bought. The refund mode is set in the right menu (swipe right) and the Pay button changes to Refund

Send receipt

When a transaction is finished, the application should display a receipt. That receipt can be sent to the customer as an PDF file via E-mail. To send the receipt the user can press the text field labeled E-mail address and a keyboard will appear. The user can enter the customer e-mail address and press the Send button. Then the application will create an e-mail message using the native iOS e-mail application, with the customers e-mail address and the receipt as an attachment. The the user can either add something to the message body or just press Send. Then the user is returned to the mPOS application.

Transactions details

View transaction details

Transaction details are shown after a transaction has been processed. Details can also be accessed through the Payments History list (swipe left on payments view). It shows important information about the transaction and from this view you can send the receipt by email. Pressing the details button brings up the receipt slip as it comes from handpoint systems.

Top

Send receipt

Enter the email address of your customer to send the receipt by email. She will get the receipt in pdf format along with the description and image (if available).
